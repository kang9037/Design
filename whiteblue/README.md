대입 성공을 위한 스마트 관리 시스템 랜딩페이지 및 회원 관리 플랫폼

## 🚀 빠른 시작

### 1. 로컬 서버 실행
```bash
# Python이 설치되어 있다면
python -m http.server 8000

# 또는 Node.js의 http-server 사용
npx http-server -p 8000

# 브라우저에서 http://localhost:8000 접속
```

### 2. 파일 열기
모든 파일이 같은 폴더에 있는지 확인 후 `index.html`을 브라우저로 열기

## 🎨 디자인 시스템

### 컬러 팔레트
```css
/* 메인 컬러 (블루) */
--primary-500: #3b82f6;
--primary-600: #2563eb;
--primary-700: #1d4ed8;

/* 그레이스케일 */
--gray-50: #fafafa;
--gray-100: #f4f4f5;
--gray-700: #3f3f46;
--gray-900: #18181b;
```

### 타이포그래피
- **폰트**: Pretendard (로컬), 맑은 고딕 (폴백)
- **제목**: 700-900 weight
- **본문**: 400-500 weight

### 레이아웃
- **컨테이너**: 1200px max-width
- **그리드**: CSS Grid + Flexbox
- **반응형 브레이크포인트**:
  - 데스크톱: 1200px+
  - 태블릿: 768px ~ 1199px
  - 모바일: ~767px

## 🎬 히어로 영상 설정

### 방법 1: 자동 생성 (추천) ⭐
1. `generate-video.html` 열기 (크롬 브라우저)
2. "녹화 시작" 버튼 클릭
3. 10초 기다리기
4. 다운로드된 `hero-video.webm`을 `assets/` 폴더에 저장
5. 완료! `index.html` 열어서 확인

### 방법 2: 영상 없이 사용
영상 파일이 없으면 자동으로 **CSS 애니메이션 배경**이 표시됩니다.

### 방법 3: 직접 영상 사용
`assets/hero-video.mp4` 또는 `.webm` 파일을 넣으면 자동으로 적용됩니다.

### 완료된 최적화
- ✅ **페이지별 CSS 분리**: 필요한 스타일만 로드 (30-65% 용량 절감)

## 🆘 문제 해결

### 스타일이 적용되지 않음
- 브라우저 콘솔에서 CSS 파일 로드 확인
- 파일 경로가 올바른지 확인
- 브라우저 캐시 삭제 (Ctrl + Shift + R)

### JavaScript 오류
- 브라우저 콘솔에서 에러 메시지 확인
- JS 파일 경로가 올바른지 확인
- 로컬 서버를 통해 실행 (파일:// 프로토콜 대신)

### 히어로 영상이 안 보임
- `assets/hero-video.mp4` 또는 `.webm` 파일이 있는지 확인
- 파일 형식이 브라우저에서 지원되는지 확인
- 영상 없이도 CSS 애니메이션 배경이 자동 표시됨
